# Risk Register — Physics.explained iOS MVP

Each risk names the phase where it bites and the prompt/phase that owns the mitigation. Review this file at the start of every phase.

| # | Risk | Bites in | Mitigation | Owner |
|---|---|---|---|---|
| R1 | **Bearer-auth change regresses production web chat.** `middleware.ts` + ask routes are live production code. | P0 | Additive only: bearer branch checked *before* the cookie branch; cookie logic byte-for-byte untouched. Acceptance includes a browser cookie-flow regression test and `curl` tests for both paths. Deploy behind normal Vercel preview → prod flow. | P0-2, P0-4 |
| R2 | **Draft content leakage.** `content_entries` has no status column; naive PostgREST reads expose drafts. | P2+ | iOS lists/navigates ONLY via the live-only taxonomy view from the P0 migration; never query `content_entries` by anything other than an exact (kind, slug, locale) already vetted by taxonomy. P0 acceptance curls a known draft slug and expects zero rows; P2 adds a deep-link test. | P0-1, P2-2 |
| R3 | **SwiftMath can't render some content TeX** (KaTeX is more permissive). | P3, P5 | P3 TeX audit: run every `formula`/`equation` string in live content through SwiftMath in a test target; failures → simplify TeX upstream in content or accept designed mono-code fallback. Chat uses the same `FormulaView`, so coverage carries to P5. | P3-1, P3-5 |
| R4 | **Streaming LaTeX re-render performance** — re-typesetting on every `text` delta janks the chat. | P5 | Fence-parse and typeset only *completed* segments; the trailing in-progress chunk renders as plain text until its closing delimiter arrives. Formula images cached by `(tex, size, color)`. Profile in P8. | P5-4, P8-2 |
| R5 | **Expression-evaluator mismatch vs mathjs** for `:::plot` fences (functions, implicit multiplication, constants). | P5 | Golden test corpus: real plot fences captured from production conversations + the web test suite; document the supported grammar; non-finite samples clipped; on parse failure render a designed "plot unavailable" card — never crash. Evaluator sits behind `ExpressionEvaluating` so a hand-rolled Pratt parser can replace the library without touching call sites. | P5-3 |
| R6 | **SSE lifecycle on cellular/app-suspend** — iOS kills sockets on background; heartbeats stop. | P5 | `waitsForConnectivity`; 60s no-event watchdog (server pings every 15s, so 60s = dead); on suspend/termination mark the message truncated with a retry affordance; honor `Retry-After` on 429; `done` is the only completion signal. | P5-2, P5-6 |
| R7 | **App Review: payments.** Any purchase UI or external payment link for digital goods without IAP = rejection (Guideline 3.1.1). | P8 review | 402 quota wall shows NO price, NO link, NO "upgrade on our website". Plan name may display in Settings as read-only account info. IAP is a separate post-MVP project. | P5-5, P8-4 |
| R8 | **App Review: Sign in with Apple (4.8) & account deletion (5.1.1(v)).** Google login without Apple login, or accounts without in-app deletion, are hard rejections. | P8 review | Apple sign-in ships in P5-1 alongside Google. Account deletion end-to-end in P8-3 (server support spec'd in `06-backend-changes.md`). Both are on the store-prep checklist. | P5-1, P8-3 |
| R9 | **Supabase Auth CAPTCHA blocks native OTP.** If dashboard CAPTCHA (Turnstile) is enabled, `signInWithOtp` from native fails. | P5 | P0 checklist verifies the dashboard setting BEFORE P5 starts. If enabled: either disable for the project or scope an alternative (Apple/Google only at MVP). Recorded in Assumption A8. | P0-4 |
| R10 | **Image URL split** — `Block` figure `src` may be a web path (`/images/...`, served from the Next.js `public/`) or a Supabase storage path. | P3 | Single `ImageURLResolver` in PXAPI with unit-tested rules (spec in `02-api-contracts.md`); web-relative paths resolve against the production site origin. Stretch: migrate repo-public content images into the storage bucket (noted in `06-backend-changes.md`, optional). | P2-2, P3-3 |
| R11 | **pbxproj corruption by the executor.** Agent edits to Xcode project files are error-prone and hard to review. | P1+ | All code in the SPM package + synchronized `App/` folder; hard rule #1 in `01-architecture.md` §8; every prompt's do-nots repeat it. Project file changes are Fable/user-only. | P1-1, all prompts |
| R12 | **`branches.ts` ↔ DB taxonomy drift** — someone edits the registry and forgets to publish. | ongoing | The publish script is the only writer of `taxonomy_*`. Add a CI-style check (script) in the web repo comparing registry to DB (part of P0-1 acceptance tooling); run it in the content-publish workflow. | P0-1 |
| R13 | **Scene visual drift vs web** — ported scenes subtly wrong (scaling, colors, physics constants). | P4, P6 | Playbook checklist per scene: side-by-side against the P0 snapshot (`snapshot_url` is the reference render); math constants come from PXMath ports with golden tests; Gallery deep-link screenshots archived per wave. | P4-\*, waves |
| R14 | **Executor scope creep / improvisation** (Opus adding dependencies, restructuring, "improving" contracts). | P2+ | Every prompt: explicit do-nots, pinned file lists, "the answer is in 02-api-contracts.md — never guess". Architecture hard rules §8. Deviations require editing the doc first, which surfaces them in review. | prompt pack |
| R15 | **App Review: AI-generated content (1.2/4.0).** Reviewers increasingly expect moderation/reporting affordances for LLM chat. | P8 review | Server already refuses injection/off-topic; add a per-message "Report" action (mailto or simple endpoint) in P8-4; App Review notes describe the physics-tutor scope, provider content policies, and the auth gate. | P8-4 |
| R16 | **SSE stall / server watchdog interplay** — server emits `error{code:"UPSTREAM_STALL"}` after 20s upstream silence; client must distinguish this from network death. | P5 | Treat `error` events as terminal-with-message (render partial + notice); treat transport EOF without `done` as truncation-with-retry. Replay-test both from fixture transcripts. | P5-2 |
| R17 | **Snapshot pipeline gaps** — some scenes need interaction/animation frames to look meaningful as stills. | P0, P3 | Snapshot script waits for first paint + one settle tick; curated list of scenes needing a custom snapshot time offset; figures with useless stills get prioritized into Wave 1 native ports. Coverage metric in P0 acceptance (≥95%). | P0-3, P6 |
