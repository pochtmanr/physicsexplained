# Architecture — Physics.explained iOS

The engineering constitution. Every prompt in `prompts/` assumes these rules; do not deviate without updating this document first. Siblings: `02-api-contracts.md` (wire contracts), `04-block-rendering-spec.md` (reader), `05-scene-porting-playbook.md` (scenes).

## 1. Repository layout

```
ios/
  project.yml                     # XcodeGen spec — the ONLY way the project file changes (regenerate, never hand-edit)
  PhysicsExplained.xcodeproj      # THIN shell, generated by `xcodegen generate --spec ios/project.yml`. Never edited by hand.
  App/                            # App target sources (~20 files max)
    PhysicsExplainedApp.swift     # @main, DI wiring
    RootView.swift                # Tab shell (Browse / Ask / Play / Settings), NavigationSplitView on iPad
    Auth/                         # Sign-in screens (Apple/Google/OTP)
    Settings/
    Gallery/                      # DEBUG-only visual harness (see §7)
  PhysicsPackages/                # ONE local Swift package, all real code lives here
    Package.swift
    Sources/
      PXFoundation/
      PXModel/
      PXMath/
      PXDesign/
      PXAPI/
      PXScenes/
      PXBlocks/
      PXAsk/
    Tests/
      PXModelTests/  PXMathTests/  PXAPITests/  PXAskTests/
```

The `.xcodeproj` references `App/` as a folder (synchronized group) and depends on the local package. All feature work = adding files inside `PhysicsPackages/Sources/<Target>/` — no project-file surgery, which is the #1 way agent sessions destroy Xcode projects (Risk R11).

## 3. External dependencies (pinned — do not add others)

| Package | Version pin | Why |
|---|---|---|
| `supabase-swift` | latest 2.x | Official; auth (Apple/Google/OTP), PostgREST, storage. |
| `GRDB.swift` | latest 7.x | Local cache. Explicit SQL beats SwiftData's implicit magic for agent-written code: schema visible in one file, migrations trivial, testable without simulator. |
| `Nuke` | latest 12.x | Image loading/disk cache for snapshots, figures, physicist photos. `LazyImage` in SwiftUI. |
| `SwiftMath` | latest | Native MathML-quality LaTeX typesetting (iosMath lineage); no WebView per formula (streaming perf, offline). |
| `swift-math-parser` (bradhowes) | latest | Runtime evaluation of `:::plot` expressions (mathjs subset) behind `ExpressionEvaluating` — swappable for a hand-rolled Pratt parser if coverage gaps appear (Risk R5). |

Adding a dependency requires editing this table first (the prompts forbid it otherwise).

## 4. Concurrency & state conventions

- **Swift 6 strict concurrency, complete checking.** No `@unchecked Sendable` without a comment justifying it.
- Network/IO = **actors**: `AskClient` is an actor exposing `AsyncThrowingStream<AskEvent, Error>`; `ContentRepository` is an actor owning the GRDB `DatabaseQueue`.
- UI state = `@MainActor @Observable` view models (one per screen: `BrowseModel`, `TopicModel`, `AskViewModel`, `PlaygroundModel`). Views stay dumb.
- Streams: the SSE actor yields typed `AskEvent`s; the view model reduces them into `MessageState` on the main actor. Cancellation = task cancellation; the actor must terminate its stream on `CancellationError` and on the 60s no-event watchdog.
- Scene animation: `TimelineView(.animation)` drives per-frame time; physics stepping happens in the scene's value-type state (no timers, no Combine).

## 5. Error handling conventions

- All API failures decode into typed errors in PXModel (`AskAPIError.quotaExhausted(reason:)`, `.rateLimited(retryAfter:)`, `.unauthenticated`, `.askDisabled`, `.badRequest(message:)`, `.server(message:)`, `.stream(code:message:)`).
- View models map typed errors to a small set of UI states: `signInRequired`, `quotaWall`, `cooldown(until:)`, `offline`, `serviceOff`, `genericRetry`. No raw error strings in UI.
- Never swallow errors: log via `os.Logger` with category = target name; degraded renders (TeX fallback, snapshot fallback) must be visually intentional per design system, not accidental.

## 6. Testing strategy

Fast loop (no simulator — seconds, run constantly):
```bash
swift test --package-path ios/PhysicsPackages
```
covers: PXModel decode fixtures (real JSON rows checked into `Tests/PXModelTests/Fixtures/`), PXMath golden values vs TS originals, SSE parser replay (recorded transcript fixtures), fence-parser port parity, expression-evaluator vs mathjs goldens, ImageURLResolver cases.

Slow loop (simulator — run at milestones):
```bash
xcodebuild -project ios/PhysicsExplained.xcodeproj -scheme PhysicsExplained \
  -destination 'platform=iOS Simulator,name=iPhone 17 Pro' build
xcrun simctl io booted screenshot /tmp/shot.png   # visual checks via Gallery deep links
```
(Adjust simulator name to what `xcrun simctl list devices available` shows on this Mac.)

UI correctness is verified visually through the **Gallery** (§7) + screenshots, not UI tests, at MVP.

## 7. Gallery (debug visual harness)

A DEBUG-only screen (App target) listing: every design token, every text style, every PXDesign component, every registered scene (grouped by branch, with live/snapshot indicator), FormulaView samples (the TeX audit corpus), and block-renderer fixtures. Supports deep links (`physicsexplained://gallery/scene/<id>`) so a prompt can say "screenshot scene X" mechanically. **Every visual PR-equivalent ends with a Gallery screenshot.** This is the executor's primary self-verification tool for anything visual.

## 8. Hard rules for the executor

1. **Never edit `PhysicsExplained.xcodeproj`** (pbxproj, schemes, plists inside it). File additions go in the SPM package or the synchronized `App/` folder.
2. All code in `PhysicsPackages/Sources/`; the App target holds only entry, navigation shell, auth screens, settings, Gallery.
3. No new external dependencies (see §3).
4. Respect the dependency flow in §2 — e.g. PXScenes must not import PXAPI.
5. Scenes never draw their own frame/caption/background — SceneCard owns all chrome.
6. **Build after every 2–3 files** (`swift build --package-path ios/PhysicsPackages` for package-only work); never batch 15 files then debug.
7. No hardcoded URLs/keys — everything through PXFoundation config.
8. Locale: always thread `AppLocale` through APIs even though MVP pins `"en"`.
9. When a contract question arises, the answer is in `02-api-contracts.md` or the referenced web source file — never guess from memory.
10. Do not modify files in the web app (outside `ios/` and `docs/ios/`) unless the prompt explicitly says so (P0 backend prompts do).
