# Physics.explained — curriculum roadmap

The full picture. Six branches. Each branch is a sequence of modules; each module is a small cluster of topics; each topic is one page on the site.

This index locks the shape so future content work can proceed without re-deciding what a "complete" physics.explained looks like. Per-branch docs (`01-…` through `06-…`) carry the punch-list detail — hook, seven-section outline, scenes, physicists, glossary — for every individual topic.

Figure numbering restarts inside each branch. § 01 runs FIG.01–FIG.31; § 02 starts FIG.01 again, and so on. This keeps the numbering stable when a branch adds or reorders modules later.

## Out of scope (for now)

The six branches above cover the canonical undergraduate physics curriculum. A few adjacent fields could become their own branches later, but aren't committed:

- **§ 07 · Condensed Matter / Solid State** — band theory, phonons, superconductivity, semiconductors, topological phases. The largest live subfield of physics research. Could fairly carry its own branch once the basics are in place; parts of it currently live inside Thermo (phase transitions, Ising) and Quantum (multi-particle systems).
- **§ 08 · Plasma Physics** — the fourth state of matter. Fusion devices, tokamaks, magnetohydrodynamics, astrophysical plasmas. A narrower field than the others, but with its own distinct tools.
- **§ 09 · Biophysics / Medical Physics** — molecular motors, imaging modalities, radiotherapy, the physics of life. Applied rather than foundational, but visually rich.

Standard "fields" that are *not* getting their own branch — they already live inside an existing one:

- **Optics** → § 02 Electromagnetism Module 8
- **Acoustics** → § 01 Classical Mechanics Module 5 (Waves)
- **Astrophysics** → split across § 04 Relativity Module 9 and § 06 Modern Physics Module 7
- **Chaos & non-linear dynamics** → sits across § 01 Module 3 (rigid bodies) and Module 7 (fluids / turbulence)

Commit to the six branches first. Add § 07+ only when the core six are substantially live and the gap is obvious from reader behaviour.

## Conventions that apply everywhere

**FIG numbering.** Per-branch. § 01 uses FIG.01–31, § 02 restarts at FIG.01. Printed on every topic as the eyebrow.

**Topic shape.** Every topic is a single page with seven sections, 2–3 interactive scenes, 10–12 minutes reading. Sections, scenes, and reading time are specified in the per-branch docs.

**Voice.** Galileo-chandelier. Every topic hook names a specific person or moment — Galileo timing a chandelier, Carnot writing his only book at 28, Aspect in Paris 1982. Kurzgesagt meets Stripe docs. Terse, no ceremony. Calculus shown when it clarifies, skipped when it obscures.

**Math rendering.** KaTeX via rehype-katex. No MathJax.

**Scene skeleton.** Copy an existing scene (`damped-pendulum-scene.tsx`, `kinematics-graph-scene.tsx`, `projectile-scene.tsx`) — all use `useAnimationFrame`, `useThemeColors`, canvas + ResizeObserver, cyan `#5BE9FF` accent, JetBrains Mono for HUD text.

**Physics functions.** Pure TypeScript modules in `lib/physics/*.ts`. One module per topic or per coherent cluster. Unit-tested via vitest. Never import React.

**Content adds.** New physicists: direct edit to `lib/content/physicists.ts` + `messages/<locale>/physicists.json`. New glossary terms: direct edit to `lib/content/glossary.ts` + `messages/<locale>/glossary.json`. New topic: append to `CLASSICAL_MECHANICS_TOPICS` (or equivalent) in `lib/content/branches.ts`, flip `status` when the page is live.

**i18n.** Every topic ships with `content.en.mdx` first. Hebrew translation (`content.he.mdx`) is the last step of each topic, wired via `HeContent` in `page.tsx`.

**Definition of done per topic.** `content.en.mdx` replaces skeleton → scenes render → physicists/glossary live with cross-links → `status: "live"` + `readingMinutes: N` in `branches.ts` → `npx tsc --noEmit` clean → manual smoke test at `/en/{branch}/{topic}` → (optional) Hebrew shipped.
